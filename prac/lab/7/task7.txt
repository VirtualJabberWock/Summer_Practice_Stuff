Задания.


1. Реализовать арифметику матриц.

Условия для оценки 3

Операции над коэффициентами реализовать как операции в поле (некотором фиксированном в завимости от типа).
Рассматриваем ситуацию больших разреженных матриц. Тогда хранить их в виде двумерного массива не получится.
Необходимо хранить матрицы в двух списках, по сути задающих индексацию по строкам и по столбцам.
Элементы матрицы с индексами хранить в некоторой структуре. На эту структуру ссылаются элементы списков.
Первый список задаёт строки. Его элементы хранят индекс строки и заголовок списка элементов одной строки. Элементы этого вложенного списка ссылаются на структуры элементов матрицы. Т.е. получается список списков для строк.
Аналогично будет список списков для столбцов.
Эти списки позволяют осуществлять обход элементов матрицы по строкам и столбцам.

Реализовать операции:
- вывод матриц
- инициализация
- сложение
- вычитание
- умножение
- определитель
- нахождение обратной матрицы

Продемонстрировать на тестовых примерах с размерами матриц в 1000000-1000000000 строк и столбцов. При этом количество элементов должно быть относительно небольшим: 1000-1000000.


Условия для оценки 4

Дополнительно.

Все операции должны быть реализованы с использованием динамического полиморфизма. Т.е. для матриц с коэффициентами из разных полей должна быть одна реализация каждой из требуемых операций, которая готова работать с матрицами над разными типами коэффициентов.

Для динамического полиморфизма необходимо конкретизировать операции над коэффициентами (указатели на эти функции и будут лежать в таблицах виртуальных функций).
Операции над коэффициентами близких типов (например, char и int) реализвывать с использованием статического полиморфизма (т.е. одинаковый код для них не должен дублироваться, а должен генерироваться из общего шаблонного кода).
